## Wire protocol (verbatim grammar)

Fields are `|`-separated. Payload is the last field so it may itself never contain `|`
(base64 alphabet `A‚ÄìZ a‚Äìz 0‚Äì9 + / =` has no `|`, so this holds).

```
TX|INIT|<fileName>|<mimeType>|<nChunks>|<chunkSize>|<salt>
TX|DATA|<seq>|<nChunks>|<salt>|<payloadBase64>
TX|END|<nChunks>|<salt>
RX|ACK_INIT|<salt>
RX|ACK|<seq>|<salt>
RX|ACK_END|<salt>
```

- `generateSalt()` = `Math.random().toString(36).substring(2, 8)` ‚Üí 6 base36 chars.
- `TX_TIMEOUT_MS = 2500`; Tx loop tick = `setInterval(txLoop, 100)`.
- QRious: `{ size: 300, level: 'M', value: 'QR_FTP_IDLE' }`.
- jsQR called with `{ inversionAttempts: 'dontInvert' }`.
- Idle sentinel value shown on the QR canvas: `QR_FTP_IDLE`.

## Sub-step 1.3 ‚Äî `app/js/tx.js` + `app/js/rx.js`

**Target**: the stop-and-wait ARQ state machines ‚Äî the behavioral core.

**`tx.js`** ‚Äî states `IDLE ‚Üí INIT ‚Üí SENDING ‚Üí END`:
- `startTransmission(file, chunkSize, {onFrame, onProgress, onDone})`:
  `FileReader.readAsDataURL` ‚Üí `base64 = result.split(',')[1]` ‚Üí char-chunks of `chunkSize`.
- `txLoop` every 100 ms: if `now - lastTxTime < 2500` do nothing; else re-roll salt, emit
  the current-state frame (INIT / DATA[currentSeq] / END).
- ACK handling: `ACK_INIT` (in INIT) ‚Üí SENDING, force immediate send; `ACK` with
  `seq===currentSeq` (in SENDING) ‚Üí `currentSeq++`, force send, ‚Üí END when done; `ACK_END`
  (in END) ‚Üí clear timer, done, show `QR_FTP_IDLE`.

**`rx.js`** ‚Äî states `WAIT_INIT ‚Üí RECEIVING ‚Üí DONE`:
- On `TX|INIT`: capture name/mime/nChunks/chunkSize, allocate `chunks[]`, always reply
  `RX|ACK_INIT` (idempotent).
- On `TX|DATA` while RECEIVING: if `seq===expectedSeq` store payload, `expectedSeq++`;
  always reply `RX|ACK|seq`.
- On `TX|END`: reply `RX|ACK_END`; if `expectedSeq===nChunks` ‚Üí reassemble
  (`atob(join(''))` ‚Üí `Uint8Array` ‚Üí `Blob(type=mime)`), expose download URL.

**Pass criterion**: driving `tx` and `rx` against each other through an in-memory frame
relay (no camera) completes a transfer and reassembles a byte-identical file; timeout
re-send fires when an ACK is withheld. (Node test with a fake FileReader/atob shim, or the
1.5 loopback shim.)

**Output log**: `working/phase1_step3_tx-rx.md`. **‚Üê 1.3 lock-in approval point.**

## Sub-step 1.5 ‚Äî End-to-end verification

**Target**: prove byte-exact round-trip.

**Method**: a loopback harness (`app/loopback.html` or a Node/jsdom driver) wires a `tx`
instance's emitted frames directly into an `rx` instance and vice-versa (ACKs back),
bypassing optics, and transfers a known test file. Compare SHA-256 of the reassembled
bytes to the source.

**Pass criterion**: `sha256(received) === sha256(source)` for at least a text file and a
small binary (e.g. PNG). Record both hashes.

**Output log**: `working/phase1_step5_e2e-verify.md`.
